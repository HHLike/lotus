# Lotus Agent 提示与通知优化设计

## 背景

Lotus 当前会把通过 Agents 面板启动的命令保存到 `sessions.json`，并在下次启动时自动写入恢复后的 PTY。因此，只要会话中保存了 `pi` 等 Agent 命令，打开 Lotus 就会再次启动 Agent 并显示其交互提示。

前端还把 `isAgent` 当作 Tab 的永久状态。Agent 结束后该状态不会重置，后续普通命令可能继续被判定为 Agent 命令。Agent 命令结束时也不检查 Tab 是否在前台或运行时长，因而桌面通知过于频繁。

## 目标

- 启动 Lotus 时只恢复 Tab、标题和工作目录，不自动重启 Agent。
- Agent 状态只覆盖当前命令的生命周期，不污染后续命令。
- 只有后台运行至少 10 秒的命令结束时才发送桌面通知。
- 运行时显示状态徽章；后台完成后保留结果徽章，用户查看 Tab 时清除。
- 保持对已有 `sessions.json` 的兼容。

## 非目标

- 不新增设置页面或通知配置项。
- 不恢复终端输出或真正的进程状态。
- 不改变 Agents 面板的主动启动流程。
- 不改变系统通知工具的实现方式。

## 会话恢复设计

`TabSession.command` 暂时保留，用于反序列化旧版 `sessions.json`，但恢复会话时忽略该字段，不再把命令加入延迟执行队列。

新会话快照不再持久化 Agent 启动命令。已有数据中的 `command` 会在 Lotus 下一次保存会话时被写为 `null`。这样无需迁移脚本，也不会因为旧字段导致启动失败。

用户仍可从 Agents 面板主动启动 Agent。该启动命令只用于当前运行，不成为下次启动时需要重放的会话状态。

恢复出的 Tab 可以保留原标题（例如 `pi`）和工作目录，但标题不表示 Agent 正在运行，也不会单独触发 Agent 运行状态。

## 前端状态设计

每个 Tab 的运行时状态只描述当前命令：

- `busy`：当前是否有命令运行。
- `cmd`：当前命令。
- `isAgent`：当前命令是否为 Agent 命令。
- 完成事件标识：用于防止同一结束事件重复通知。

命令开始时更新上述状态。如果命令是 Agent，则添加 Agent 运行样式；否则移除旧的 Agent 运行样式。

命令结束时清除 `busy`、`cmd` 和 `isAgent`，并移除 Agent 运行样式。标题匹配 Agent 名称本身不再把 Tab 永久标记为 Agent。

## 通知与徽章策略

### 命令开始

- 显示运行中徽章。
- 记录当前命令类型。

### 命令结束

- 若 Tab 当前处于前台：清除状态徽章，不发送桌面通知。
- 若 Tab 在后台：显示完成或失败徽章。
- 仅当后台命令的 `duration_ms >= 10000` 时发送一次桌面通知。
- Agent 与普通命令遵循相同通知门槛；通知标题可以继续区分 Agent 与普通命令。
- 缺少或无效运行时长时不发送通知。

### 查看 Tab

切换到某个 Tab 时清除它的完成或失败徽章。若命令仍在运行，则保留运行中徽章。

### 重复事件

前端记录最近处理的结束事件标识。同一个命令结束事件如果被重复投递，只更新一次 UI，并且最多发送一次通知。新的命令开始时重置该标识。

## 数据流

1. Shell integration 上报命令开始。
2. Rust 后端记录开始时间并发送 `command_started`。
3. 前端设置当前命令状态并显示运行中徽章。
4. Shell integration 上报命令结束。
5. Rust 后端计算运行时间并发送 `command_finished`。
6. 前端根据当前 Tab、运行时长和重复事件标识决定徽章与通知。
7. 用户打开后台 Tab 后清除结果徽章。

## 错误处理

- 旧会话文件包含 `command` 时正常读取，但不执行。
- 无法保存会话时沿用现有日志告警，不影响终端使用。
- `notify-send` 不存在或执行失败时继续静默跳过。
- 缺少运行时长、未知 Tab 或重复完成事件都不得产生额外通知。

## 测试与验证

- 单元测试会话数据兼容：旧 `command` 字段可读取，新快照不保留可重放命令。
- 验证恢复流程不会生成待执行 Agent 命令。
- 验证前台完成不通知且不保留完成徽章。
- 验证后台运行不足 10 秒不通知。
- 验证后台运行至少 10 秒只通知一次，并保留结果徽章。
- 验证切换到结果 Tab 后清除徽章。
- 验证 Agent 完成后普通命令不会被误判为 Agent。
- 运行 Rust 测试、编译检查，以及适用于前端通知策略的自动化检查。

## 兼容性

现有配置文件无需变化。`sessions.json` 的字段结构保持可读兼容，旧版 Lotus 仍可读取 `command: null`。本次变更只收紧恢复、状态和通知行为。
